## Guidelines

Colours are one of the most obvious ways to convey meaning in design. Depending on its context of use and on the amount of time we see them, we start to associate colour with meaning. Each of the colours used in the Workleap ecosystem complement each other and has a specific role to play.

### Communication is key

Although we value an aesthetically pleasing use of colour, we place a higher value on clear communication. Colour supports the purpose of the content, communicating things like hierarchy, interactive states and differences between visual elements.

### Colours have meaning

Each colour has assigned sentiment based on how they function within the interface. Defined colour roles make things easy to modify and customize later. Their meaning is also expanded to all Workleap verticals so that users understand that theyâ€™re in the same ecosystem and can recognize its codes.

#### Deprecation notice

Tokens with a `-active` suffix are deprecated and should not be used in new designs. They will be removed in a future release, prefer using their `-press` counterparts. In some instances `-active` tokens should be replaced with a `-selected` token, this should be documentend in your Figma files, in doubt validate with a designer.
